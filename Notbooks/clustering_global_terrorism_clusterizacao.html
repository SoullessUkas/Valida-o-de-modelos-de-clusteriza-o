# Clula corrigida
df = pd.read_csv('globalterrorismdb_0718dist.csv', encoding='ISO-8859-1', low_memory=False)

print(f'Linhas e colunas carregadas: {df.shape}')

# Remove eventid se existir
if 'eventid' in df.columns:
    df = df.drop(columns=['eventid'])

# Seleciona colunas
cols = ['iyear', 'imonth', 'iday', 'extended', 'region']
df = df[cols].copy()

# Tratamento de Nulos e Zeros
df['imonth'] = df['imonth'].replace(0, np.nan)
df['iday'] = df['iday'].replace(0, np.nan)

# Preenchimento de nulos com mediana para datas
for c in ['iyear', 'imonth', 'iday']:
    df[c] = df[c].fillna(df[c].median())

# Preenchimento de extended e region
df['extended'] = df['extended'].fillna(0)
mode_region = df['region'].mode(dropna=True)[0] if df['region'].notna().any() else 0
df['region'] = df['region'].fillna(mode_region)

# Definio das features
numeric_features = ['iyear', 'imonth', 'iday', 'extended']
categorical_features = ['region']

# Pipeline de transformao
preprocessor = ColumnTransformer(
    transformers=[
        ('num', StandardScaler(), numeric_features),
        # force_int_category=False garante compatibilidade
        ('cat_region', OneHotEncoder(handle_unknown='ignore', sparse_output=False), categorical_features), 
    ],
    remainder='drop'
)

# Aplicar transformao
# Nota: sparse_output=False no OneHotEncoder j retorna array denso,
# no precisa de toarray(), mas mantivemos a lgica caso mude verses
X_processed = preprocessor.fit_transform(df)

# Garantia de formato array
if issparse(X_processed):
    X = X_processed.toarray()
else:
    X = X_processed

print(f'Dimenso aps transformao: {X.shape}')

Interpretao

  • K-Means tende a formar grupos de forma aproximadamente convexa, equilibrados quando a varincia semelhante entre dimenses.
  • DBSCAN identifica regies densas e marca pontos esparsos como rudo (-1), suportando formas no convexas.
  • Diferenas esperadas: K-Means sempre retorna k clusters; DBSCAN pode retornar nmero varivel e rudo.
  • Silhueta maior indica maior separao mdia entre clusters e menor sobreposio; Davies-Bouldin menor e Calinski-Harabasz maior so desejveis.
  • A presena de rudo e clusters com tamanhos/densidades distintos pode reduzir a Silhueta do DBSCAN quando calculada apenas em pontos no-rudo.
1. A silhueta  o ndice indicado para escolher o nmero de clusters para o algoritmo de DBScan?

Resposta: No.

Justificativa: O ndice de Silhueta assume que os clusters so convexos (redondos/esfricos) e compactos. O DBSCAN baseado em densidade e pode criar clusters de formatos complexos (como luas crescentes ou anis) que, embora corretos para a densidade, teriam uma pontuao de silhueta ruim.

O que usar: O DBSCAN no pede o "nmero de clusters" a priori, ele descobre isso sozinho baseado nos parmetros eps e min_samples. Para avaliar a qualidade do DBSCAN, prefere-se o ndice DBCV (Density-Based Clustering Validation) ou a inspeo visual do grfico de atingibilidade (Reachability Plot).

2. Um determinado problema, apresenta 10 sries temporais distintas. Gostaramos de agrup-las em 3 grupos, de acordo com um critrio de similaridade, baseado no valor mximo de correlao cruzada entre elas. Descreva em tpicos todos os passos necessrios.

Pr-processamento: Normalizar as sries temporais (ex: Z-Score) para que diferenas de amplitude (escala) no afetem a correlao. Garantir que todas tenham a mesma frequncia de amostragem.
Clculo da Correlao Cruzada: Para cada par de sries (A e B), calcular a funo de correlao cruzada (CCF) deslizando uma srie sobre a outra (lags). Extrao do Mximo: Identificar o valor absoluto mximo da correlao encontrado em qualquer "lag" (atraso). Este valor representa a similaridade mxima, independentemente de uma srie estar atrasada em relao outra. Matriz de Distncia: Construir uma matriz $10 \times 10$. Como algoritmos de cluster usam distncia (e no similaridade), converta o valor: $Distncia = 1 - Correlao\_Mxima$.Clusterizao: Aplicar o algoritmo de agrupamento sobre esta matriz de distncias.

2.1. Qual algoritmo de clusterizao voc usaria? Justifique.

Algoritmo: Clusterizao Hierrquica Aglomerativa (Hierarchical Clustering).

(Tamanho): Para um dataset minsculo (10 sries), o Hierrquico excelente e permite visualizao clara via Dendrograma. (Matriz Pr-computada): Este algoritmo trabalha nativamente recebendo uma matriz de distncias customizada (criada no passo anterior) como entrada, sem precisar das coordenadas originais dos pontos. (Definio de K): O problema pede explicitamente 3 grupos. No dendrograma, basta fazer um "corte" na altura que gere 3 ramos.

4. Sugira outra estratgia para medir a similaridade entre sries temporais.

Estratgia: DTW (Dynamic Time Warping).Passos necessrios:Matriz de Custo Local: Criar uma matriz onde cada clula $(i, j)$ representa a distncia (euclidiana) entre o ponto $i$ da srie A e o ponto $j$ da srie B.Clculo do Caminho timo: Encontrar o caminho contguo que vai do incio ao fim das sries minimizando a soma das distncias, permitindo passos diagonais, verticais ou horizontais (isso "deforma" o tempo para alinhar picos e vales que no ocorrem no mesmo instante exato).Distncia Final: A soma dos custos ao longo desse caminho timo a distncia DTW. Sries com formato similar, mas velocidades diferentes, tero baixo DTW (alta similaridade).
